## 🎯 品質檢查概述

### 檢查時機

- Part A、Part B、附錄各完成後

## 📝 檢查清單 5.1：格式規範檢查（v3.4.0 更新）

### Part A 格式檢查

#### 報告開頭
```yaml
- [ ] 包含「分析期間」標註
- [ ] 包含「數據截止」標註
- [ ] 時間格式正確（YYYY/MM/DD）
- [ ] 版本號標註為 v3.4.0 或更高
```

#### 重要訊息
```yaml
- [ ] 字數 80-120字
- [ ] 單一段落格式（非條列式）
- [ ] 包含：股價、目標價、空間、論點、風險
```

#### 評論及分析 - 子章節1：營運表現（v3.4.0 新增）
```yaml
結構檢查:
  - [ ] 總字數 300-500字
  - [ ] 標題為自然語言（非「營運表現分析」等制式標題）
  - [ ] 段落數 2-4段（建議3段）

內容完整性（v3.4.0 新增）:
  - [ ] 涵蓋核心財務表現（營收、利潤、EPS）
  - [ ] 涵蓋重大戰略調整（若有 M&A、重組）
  - [ ] 涵蓋管理層財測指引

彈性處理:
  - [ ] 若無戰略調整，是否合理擴充其他內容
  - [ ] 段落結構符合數據可得性
```

#### 評論及分析 - 子章節2：市場展望（v3.4.0 新增）
```yaml
結構檢查:
  - [ ] 總字數 300-500字
  - [ ] 標題為自然語言

內容完整性（v3.4.0 新增）:
  - [ ] 包含營收增長主力分析
  - [ ] 包含競爭分析（對手、優勢、地位）
  - [ ] 包含季度變化比較（QoQ）
    - 本季 vs 上季市場狀況變化
    - 競爭格局變化
    - 公司地位變化

季度變化檢查:
  - [ ] 若有 QoQ 數據，是否清楚比較
  - [ ] 若無 QoQ 數據，是否標註說明
  - [ ] QoQ 描述與財務數據吻合
```

#### 評論及分析 - 子章節3：估值與目標價（v3.4.0 新增）
```yaml
結構檢查:
  - [ ] 總字數 300-500字
  - [ ] 標題為自然語言

內容完整性（v3.4.0 新增）:
  - [ ] 包含 EPS 推估依據
    - 營收增長率假設
    - 營運效率/淨利率假設
  - [ ] 包含目標本益比設定理由
  - [ ] 包含 12 個月目標價（樂觀/保守）

券商預測表格（v3.4.0 新增）:
  - [ ] 是否包含券商預測彙整表格
  - [ ] 表格格式正確（券商|EPS|目標價|日期）
  - [ ] 欄位對齊正確
  - [ ] 資料來源標註（如 Morningstar）
  - [ ] 發布日期格式統一（YYYY/MM/DD）
  - [ ] 若少於10家，是否說明原因

資料來源標註:
  - [ ] 明確說明「華爾街共識」或「券商中位數」
  - [ ] 註明數據截止日期
```

#### 投資建議
```yaml
- [ ] 字數 250-300字
- [ ] 2段結構
- [ ] 第1段：估值邏輯（150-180字）
- [ ] 第2段：評等建議（100-120字）
- [ ] 無理論術語（DCF、Gordon等）
```

#### 投資風險
```yaml
風險數量:
  - [ ] 符合不確定性評級
    - Low/Medium: 3上行+3下行
    - High: 4上行+5下行
    - Very High: 4上行+6下行

每項風險:
  - [ ] 字數 120-150字
  - [ ] 標題為自然語言（非編號）
  - [ ] 包含：情境、影響、量化、時間框架
```

#### 數據來源聲明
```yaml
- [ ] 100字以內
- [ ] 包含「分析期間」
- [ ] 包含「數據截止」
- [ ] 包含所有數據類型
- [ ] 註明完整參考文獻在附錄
```

#### 總體檢查
```yaml
- [ ] Part A 總字數 3,000-3,500字（±10%）
- [ ] Markdown 格式正確
- [ ] 標題層級正確（## 和 ###）
- [ ] 中英文標點符號正確
```

### 數據一致性檢查（v3.4.0 新增）

```yaml
券商預測一致性:
  - [ ] 估值章節表格中的目標價
  - [ ] 投資建議中提到的目標價
  - [ ] Part B 價格表中的目標價
  - [ ] 三者必須一致或有明確說明

季度變化一致性:
  - [ ] 市場展望中的 QoQ 描述
  - [ ] 營運表現中的財務數據
  - [ ] 兩者描述方向一致（如都指向增長）

分析視角應用:
  - [ ] 階段0C選定的3-5個視角
  - [ ] 在三個子章節中都有體現
```

## ⚠️ 檢查清單 5.3：特殊情況處理（v3.4.0 新增）

### 數據不足處理檢查

```yaml
券商預測不足:
  - [ ] 若少於10家，表格下方是否註明
  - [ ] 註明範例：「註：Morningstar 僅提供 X 家主要券商預測」

  - [ ] 若無 QoQ 數據，是否標註說明

戰略調整缺失:
  - [ ] 若無重大 M&A 或重組，營運表現是否調整段落結構
  - [ ] 總字數仍維持 300-500字
```

### 彈性處理確認

```yaml
營運表現段落:
  - [ ] 若採用三段落，結構合理
  - [ ] 若採用二段落/四段落，是否因數據限制
  - [ ] 總字數符合要求

券商預測表格:
  - [ ] 若僅3-5家券商，是否仍以表格呈現
  - [ ] 若完全無券商數據，是否改用文字說明
```

## 📞 常見檢查失敗原因（v3.4.0 更新）

### 新增格式項目失敗

**問題1：券商預測表格缺失**  
原因：忘記新增 v3.4.0 要求  
解決：在估值章節補充券商預測表格

**問題2：季度變化分析缺失**  
原因：未注意 v3.4.0 新增要求  
解決：在市場展望補充 QoQ 分析，或標註數據不可得

**問題3：營運表現未涵蓋三方面**  
原因：僅寫財務數據  
解決：補充戰略調整和管理層指引，或調整段落分配

### 數據一致性失敗

**問題：券商預測目標價不一致**  
原因：估值章節與投資建議使用不同數據  
解決：統一使用同一來源的目標價，或明確說明差異原因
