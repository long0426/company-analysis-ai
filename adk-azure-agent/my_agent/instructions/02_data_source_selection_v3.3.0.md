## 🎯 階段0A：工具與資料源識別

### 目的

根據可用工具動態選擇最佳數據來源，確保：
- ✅ 使用最可靠的數據來源
- ✅ 數據來源有明確優先級
- ✅ 數據源變更時報告結構不變
- ✅ 所有數據可追溯
- ✅ 時間範圍處理邏輯清晰（v3.3.0 新增）

## 🎯 步驟 0A.2：選擇數據來源優先級

### 數據類型優先級矩陣

| 數據類型 | 首選來源 | 次選來源 | 最低要求 |
|---------|---------|---------|---------|
| **分析師研究報告** | 專業投資研究機構<br>(Morningstar, Bloomberg) | 券商研究 | 公司官網IR |
| **公允價值估計** | 分析師報告<br>(單一分析師深度分析) | 共識估計<br>(多位分析師平均) | 自行建模<br>(需明確標註) |
| **財務數據** | 專業財經數據庫 | 公司季報/年報 | 公司季報/年報 |
| **股價與交易數據** | 實時行情商 | 財經數據庫 | 交易所官網 |
| **評級數據** | 分析師評級 | 信評機構 | - |
| **產業分析文章** | 專業財經媒體<br>(Morningstar, Bloomberg) | 產業研究機構 | 網路搜索 |

### 選擇邏輯流程圖

```
FOR 每個所需數據類型:
  │
  ├─ IF 首選來源可用 AND 覆蓋目標公司:
  │   └─→ 使用首選來源
  │       └─→ 標註: "數據來自 [首選來源]"
  │
  ├─ ELIF 次選來源可用:
  │   └─→ 使用次選來源
  │       └─→ 標註: "數據來自 [次選來源]（首選來源不可得）"
  │
  └─ ELSE:
      └─→ 使用最低要求來源
          └─→ 標註: "數據來自 [最低要求來源]，數據有限"
```

## ⏰ 步驟 0A.4：時間篩選邏輯（v3.3.0 新增）

### 三個時間概念定義

```yaml
時間概念:
  1. 分析期間 (Analysis Period):
     起始: ANALYSIS_START_DATE
     結束: ANALYSIS_END_DATE
     用途: 定義報告分析的時間範圍
     範例: "本報告分析 AMD 於 2024年 的表現"
  
  2. 數據截止日期 (Data Cutoff Date):
     定義: 通常等於 ANALYSIS_END_DATE
     用途: 所有數據的時間上限
     範例: "所有數據截至 2024/12/31"
  
  3. 報告日期 (Report Date):
     定義: REPORT_DATE
     用途: 報告撰寫/發布日期
     範例: "本報告撰寫於 2025/01/05"

時間邏輯關係:
  ANALYSIS_START_DATE ≤ ANALYSIS_END_DATE ≤ REPORT_DATE
```

### 不同數據類型的時間處理規則

#### A. 股價與交易數據

```yaml
股價收盤價:
  取得時間點: ANALYSIS_END_DATE
  邏輯: 使用分析期間結束日的收盤價
  
  範例:
    分析期間結束: 2024/12/31
    → 取得 2024/12/31 的收盤價
  
  標註格式:
    收盤價: US$198.11
    來源: Morningstar Direct Database
    數據ID: OS065
    分析期間: 2024/01/01 - 2024/12/31
    數據截止日期: 2024/12/31
    查詢時間: 2025/01/05 10:00 UTC

52週高低點:
  計算方式: 固定為「過去52週」（向前推算52週）
  邏輯: 不跟隨分析期間調整，保持計算一致性
  
  範例:
    查詢時間: 2025/01/05
    → 計算 2024/01/06 - 2025/01/05 的52週高低
  
  註記: 如需要分析期間內的高低點，另外標註

市值:
  計算基準: ANALYSIS_END_DATE 的股價 × 流通股數
  
平均成交量:
  計算期間: ANALYSIS_END_DATE 前3個月
  
  範例:
    分析期間結束: 2024/12/31
    → 計算 2024/10/01 - 2024/12/31 的3個月平均
```

#### B. 財務數據

```yaml
財務數據處理邏輯:
  規則: 取「截至 ANALYSIS_END_DATE 已發布的最新財報」
  
  情境1（常見）: 分析期間結束後，對應季度財報尚未發布
    範例:
      分析期間結束: 2024/12/31
      Q4 2024 財報發布: 2025/01/31
      當前時間: 2025/01/05（早於財報發布）
    
    處理:
      → 使用 Q3 2024 財報（最新已發布）
      → 標註: "財務數據截至 Q3 2024（截至分析期間結束日，Q4數據尚未發布）"
  
  情境2: 分析期間結束很久後才撰寫報告，對應財報已發布
    範例:
      分析期間結束: 2024/12/31
      Q4 2024 財報發布: 2025/01/31
      當前時間: 2025/02/15（晚於財報發布）
    
    處理:
      → 仍使用 Q3 2024 財報（保持分析期間的一致性）
      → 或使用 Q4 2024 財報，但明確標註「財報發布於分析期間後」
      → 建議: 優先使用分析期間內的數據，除非特別說明
  
  情境3: 歷史分析
    範例:
      分析期間: 2023/01/01 - 2023/12/31
      報告日期: 2025/01/05
    
    處理:
      → 使用 2023/12/31 前最後發布的財報
      → 通常是 Q3 2023 財報

標註格式:
  財務數據: Q3 2024
  營收: US$6,819 百萬
  來源: Morningstar Direct Database
  分析期間: 2024/01/01 - 2024/12/31
  財報截至: 2024/09/30
  財報發布: 2024/10/29
  查詢時間: 2025/01/05 10:00 UTC
  
  註記: "Q4 2024 財報於 2025/01/31 發布，超出分析期間"
```

### 時間處理流程圖

```
FOR 每個數據類型:
  │
  ├─ IF 數據類型 = 股價/交易數據:
  │   └─→ 取得 ANALYSIS_END_DATE 時間點的數據
  │       └─→ 標註: 分析期間、數據截止日期
  │
  ├─ ELIF 數據類型 = 財務數據:
  │   └─→ 取得「截至 ANALYSIS_END_DATE 已發布的最新財報」
  │       └─→ IF 對應季度財報尚未發布:
  │           └─→ 使用前一季度財報
  │           └─→ 標註: "QX 財報於分析期間結束後發布"
  │
  ├─ ELIF 數據類型 = 分析師報告/評級:
  │   └─→ 取得「ANALYSIS_END_DATE 前最新發布的報告」
  │       └─→ 忽略超出分析期間的更新報告
  │       └─→ 標註: 報告發布日期、分析期間
  │
  └─ ELIF 數據類型 = 新聞/文章:
      └─→ 篩選「ANALYSIS_START_DATE ~ ANALYSIS_END_DATE」區間內
          └─→ 嚴格時間篩選
          └─→ 標註: 發布日期、分析期間
```

### 時間邏輯驗證檢查

```yaml
執行時檢查:
  - [ ] ANALYSIS_START_DATE ≤ ANALYSIS_END_DATE ✅
  - [ ] ANALYSIS_END_DATE ≤ REPORT_DATE ✅
  - [ ] 所有股價數據時間點 = ANALYSIS_END_DATE ✅
  - [ ] 所有財報發布日 ≤ ANALYSIS_END_DATE（或已標註例外）✅
  - [ ] 所有分析師報告發布日 ≤ ANALYSIS_END_DATE ✅
  - [ ] 所有新聞文章發布日在分析期間內 ✅

錯誤處理:
  IF 檢查未通過:
    → 系統提示錯誤
    → 要求修正時間邏輯
    → 提供修正建議
```

## ⚠️ 數據缺失處理原則（v3.3.0 更新）

### 處理流程

```
數據不可得？
  ↓
是
  ↓
優先級 = P0 (必須收集)?
  ↓
├─ 是 → 檢查是否為時間因素（v3.3.0 新增）
│       ↓
│       ├─ 是（分析期間結束後才發布）
│       │   → 使用最新已發布數據
│       │   → 明確標註原因
│       │
│       └─ 否 → 嘗試次選來源 → 找到? → 使用並標註來源
│                                 ↓
│                                否 → 嘗試最低要求來源
│                                       ↓
│                                      找到? → 使用並標註「數據有限」
│                                       ↓
│                                      否 → 標註 "N/A" + 說明原因
│
└─ 否 → 標註 "N/A" 即可
```

### 標註規範（v3.3.0 更新）

#### ✅ 正確的處理方式

```markdown
# 範例1：時間因素導致數據缺失（v3.3.0）
**財務數據**: Q3 2024
**營收**: US$6,819 百萬
**說明**: 截至分析期間結束日（2024/12/31），Q4 2024 財報尚未發布
（預計 2025/01/31 發布）。本報告使用 Q3 2024 最新已發布數據。

---

# 範例2：一般數據缺失
| 項目 | 數值 |
|------|-----:|
| 收盤價 (US$) | 198.11 |
| 52週高價 (US$) | 264.35 |
| 機構持有比例 (%) | N/A |

**註**：部分數據因數據源限制不可得。

---

# 範例3：分析師評級時間說明（v3.3.0）
**公平價值估計**: US$270.00
**分析師評級**: 買進
**評級日期**: 2024/11/12
**說明**: 使用截至分析期間結束日（2024/12/31）的最新分析師評估。
2025/01/10 發布的更新報告未納入，以保持分析期間一致性。
```

#### ❌ 錯誤的處理方式

```markdown
# 錯誤 1: 留空白
| 項目 | 數值 |
|------|-----:|
| 收盤價 (US$) | 198.11 |
| 52週高價 (US$) | 264.35 |
| 機構持有比例 (%) |  |  ← 錯誤：應標註 N/A

---

# 錯誤 2: 使用超出分析期間的數據而不說明（v3.3.0）
**財務數據**: Q4 2024
（分析期間結束 2024/12/31，但 Q4 財報 2025/01/31 才發布）
← 錯誤：應使用 Q3 2024 或明確說明使用未來數據的理由

---

# 錯誤 3: 使用估計值但不標註
**公平價值估計**: US$150.00
（實際上是自己估算，但未說明）

---

# 錯誤 4: 刪除該項目
根本不列出「機構持有比例」這個欄位
```

### 數據缺失說明位置

```yaml
輕微缺失 (1-2個數據點):
  位置: 表格下方簡單註記
  範例: "**註**: 部分數據因數據源限制不可得。"
```
