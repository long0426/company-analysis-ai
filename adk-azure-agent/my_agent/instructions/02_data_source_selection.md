## 🎯 階段0A：工具與資料源識別

### 目的

根據可用工具動態選擇最佳數據來源，確保：
- ✅ 使用最可靠的數據來源
- ✅ 數據來源有明確優先級
- ✅ 數據源變更時報告結構不變
- ✅ 所有數據可追溯

## 📊 步驟 0A.1：盤點可用工具

### 工具識別清單

```yaml
執行步驟:
  1. 列出當前環境中所有可用的數據工具
```

## 🎯 步驟 0A.2：選擇數據來源優先級

### 數據類型優先級矩陣

| 數據類型 | 首選來源 | 次選來源 | 最低要求 |
|---------|---------|---------|---------|
| **分析師研究報告** | 專業投資研究機構<br>(Morningstar, Bloomberg) | 券商研究 | 公司官網IR |
| **公允價值估計** | 分析師報告<br>(單一分析師深度分析) | 共識估計<br>(多位分析師平均) | 自行建模<br>(需明確標註) |
| **財務數據** | 專業財經數據庫 | 公司季報/年報 | 公司季報/年報 |
| **股價與交易數據** | 實時行情商 | 財經數據庫 | 交易所官網 |
| **評級數據** | 分析師評級 | 信評機構 | - |
| **產業分析文章** | 專業財經媒體<br>(Morningstar, Bloomberg) | 產業研究機構 | 網路搜索 |

### 選擇邏輯流程圖

```
FOR 每個所需數據類型:
  │
  ├─ IF 首選來源可用 AND 覆蓋目標公司:
  │   └─→ 使用首選來源
  │       └─→ 標註: "數據來自 [首選來源]"
  │
  ├─ ELIF 次選來源可用:
  │   └─→ 使用次選來源
  │       └─→ 標註: "數據來自 [次選來源]（首選來源不可得）"
  │
  └─ ELSE:
      └─→ 使用最低要求來源
          └─→ 標註: "數據來自 [最低要求來源]，數據有限"
```

#### 範例 1: Morningstar 完整覆蓋

## 📝 步驟 0A.3：數據源標註規範

### 在報告中的標註方式

#### Part A 開頭標註

```markdown
# {COMPANY_FULL_NAME} ({TICKER}) - {YEAR}年度分析

**分析師**: {ANALYST_NAME}
**報告日期**: {REPORT_DATE}
**評等**: {RATING}
**目標價**: {TARGET_PRICE}
**資料來源**: {DATA_SOURCE} 數據服務與研究報告

---
```

#### Part A 結尾：數據來源聲明（簡版）

```markdown
## 數據來源聲明

本報告所有數據與分析均來自 **{DATA_SOURCE}**：
- **財務數據**：截至 YYYY/MM/DD
- **分析師研究報告**：{來源}（最新更新：YYYY/MM/DD）
- **公平價值估計**：US$XXX.XX（{來源}，YYYY/MM/DD）
- **經濟護城河評級**：{評級}（如有）
- **不確定性評級**：{評級}（如有）
- **市場數據**：{數據庫}（YYYY/MM/DD）

**完整參考文獻清單請見報告末尾附錄**。
```

## 🔍 數據驗證與標註格式

### 每個數據點的標註要求

```yaml
數據點標註格式:
  
  收盤價: US$198.11
  來源: Morningstar Direct Database
  數據ID: OS065 (Datapoint ID)
  截止日期: 2025/12/17
  查詢時間: 2025/12/18 10:00 UTC
  
  ---
  
  公允價值估計: US$270.00
  來源: Morningstar 分析師研究報告
  Investment ID: 0P000002CH
  分析師: Abhinav Davuluri
  發布日期: 2025/11/12
  查詢時間: 2025/12/18 10:00 UTC
```

### 標註完整性檢查清單

```yaml
每個數據點必須包含:
  - [ ] 數據名稱明確
  - [ ] 數值已填入
  - [ ] 來源已標註（具體名稱）
  - [ ] 截止日期已記錄（YYYY/MM/DD 格式）
  - [ ] 查詢時間已記錄（UTC 時區）
  - [ ] 數據ID已記錄（如適用）

可選但建議包含:
  - [ ] 數據提供者（如分析師姓名）
  - [ ] 計算方法（如為計算值）
  - [ ] 數據更新頻率
```

## ⚠️ 數據缺失處理原則

### 處理流程

```
數據不可得？
  ↓
是
  ↓
優先級 = P0 (必須收集)?
  ↓
├─ 是 → 嘗試次選來源 → 找到? → 使用並標註來源
│                          ↓
│                         否 → 嘗試最低要求來源
│                                ↓
│                               找到? → 使用並標註「數據有限」
│                                ↓
│                               否 → 標註 "N/A" + 說明原因
│
└─ 否 → 標註 "N/A" 即可
```

### 標註規範

#### ✅ 正確的處理方式

```markdown
| 項目 | 數值 |
|------|-----:|
| 收盤價 (US$) | 198.11 |
| 52週高價 (US$) | 264.35 |
| 機構持有比例 (%) | N/A |

**註**：部分數據因數據源限制不可得。
```

```markdown
**公平價值估計**: N/A
**說明**: Morningstar 未覆蓋本公司，無分析師公允價值估計。
本報告使用自建模型估算目標價為 US$XX.XX（詳見投資建議章節）。
```

#### ❌ 錯誤的處理方式

```markdown
# 錯誤 1: 留空白
| 項目 | 數值 |
|------|-----:|
| 收盤價 (US$) | 198.11 |
| 52週高價 (US$) | 264.35 |
| 機構持有比例 (%) |  |  ← 錯誤：應標註 N/A

---

# 錯誤 2: 使用估計值但不標註
**公平價值估計**: US$150.00
（實際上是自己估算，但未說明）

---

# 錯誤 3: 刪除該項目
根本不列出「機構持有比例」這個欄位
```

### 數據缺失說明位置

```yaml
輕微缺失 (1-2個數據點):
  位置: 表格下方簡單註記
  範例: "**註**: 部分數據因數據源限制不可得。"

中度缺失 (3-5個數據點):
  位置: Part A 數據來源聲明
  範例: "**數據限制**: 由於專業投資研究機構未覆蓋本公司，
         機構持有比例、共識預估等數據不可得。"

嚴重缺失 (>5個數據點):
  位置: 
    1. Part A 開頭明確提示
    2. Part A 數據來源聲明詳細說明
    3. 附錄專門章節說明數據限制
```
