## 🎯 四大核心原則

### 原則 1：理論在內，自然在外

**定義**：使用理論框架指導分析邏輯，但報告中使用自然語言表達

**實踐方法**：
```
內部思考（不呈現）: 
"根據 Porter 五力模型，分析競爭格局..."

外部表達（呈現給用戶）:
"Broadcom 利用交換晶片搭售策略，可能侵蝕 Marvell 市佔率..."
```

**核心要求**：
- ✅ 分析深度符合理論框架要求
- ✅ 邏輯嚴謹，符合學術標準
- ✅ 報告中零理論術語
- ✅ 一般讀者可理解，專業讀者感到深度

---

### 原則 2：因地制宜

**定義**：根據產業與公司特性選擇適合的分析視角，不生搬硬套

**實踐方法**：

```yaml
半導體產業:
  必選視角: 供需分析、技術替代風險
  不適用: 訂閱模式分析、網路效應

軟體SaaS產業:
  必選視角: 訂閱模式健康度、網路效應
  不適用: 供需分析、庫存週轉

金融服務產業:
  必選視角: 監管政策風險、利率週期影響
  不適用: 技術替代、供應鏈
```

**核心要求**：
- ✅ 每個產業有專屬分析框架
- ✅ 視角選擇有明確理由
- ✅ 不使用不適用的分析方法
- ✅ 特殊情況有觸發器機制

---

### 原則 3：透明可溯源

**定義**：提供完整的數據來源與引用資訊，確保可追溯性

**實踐方法**：

```markdown
Part A 開頭：
**資料來源**: Morningstar 數據服務與研究報告

Part A 結尾：
**數據來源聲明**
本報告所有數據與分析均來自 Morningstar：
- 財務數據：截至 2025/12/17
- 分析師研究報告：Morningstar（最新更新：2025/11/12）

附錄：
**完整參考文獻清單**
1. Morningstar 分析師研究報告
   - Investment ID: 0P000002CH
   - 報告標題: AMD: Strong AI GPU Growth...
   - 發布日期: 2025/11/12
```

**核心要求**：
- ✅ 每個數據有來源標註
- ✅ 每個數據有截止日期
- ✅ 附錄提供完整引用清單
- ✅ 可重複驗證

---

### 原則 4：資料源中立

**定義**：根據可用工具動態選擇最佳數據來源，不綁定特定平台

**實踐方法**：

```yaml
數據類型優先級矩陣:
  分析師研究報告:
    首選: 專業投資研究機構 (Morningstar, Bloomberg)
    次選: 券商研究
    最低: 公司官網IR
  
  股價數據:
    首選: 實時行情商
    次選: 財經數據庫
    最低: 公司官網

選擇邏輯:
  IF 首選來源可用且覆蓋目標公司:
    使用首選來源
  ELIF 次選來源可用:
    使用次選來源，並在報告中註明
  ELSE:
    使用最低要求來源，並標註「數據有限」
```

**核心要求**：
- ✅ 支援多種數據源
- ✅ 有明確的優先級邏輯
- ✅ 數據源變更時報告結構不變
- ✅ 在報告中標註實際使用的數據源

```yaml
  ❌ "根據我所知"
  ❌ "Looking at..."

```

## ✅ 推薦的自然表達方式

### 替代方案對照表

| 禁用表達 | 推薦表達 |
|---------|---------|
| 根據 Porter 五力模型分析競爭格局 | 分析產業競爭格局發現... |
| 運用 DCF 模型估值 | 目標價基於未來盈利預期與合理估值倍數 |
| 依據護城河理論 | 公司建立的競爭優勢包括... |
| 供需分析顯示 | 市場供給持續緊張，需求保持強勁 |
| 根據資本配置效率理論 | 公司資本運用效率優異，投資回報率持續高於資金成本 |

### 自然語氣範例

#### ✅ 好的範例

```markdown
AMD 在 AI GPU 市場取得突破性進展，2024年 AI GPU 營收達
US$50億，從2023年的幾乎零營收實現跨越式增長。公司與
OpenAI 建立的戰略合作夥伴關係成為重要轉折點。

大型雲端服務商基於供應鏈韌性與議價能力考量，積極尋求第二
供應來源。然而，AMD 仍需大幅提升其 ROCm 軟體平台的成熟度，
才能真正挑戰 Nvidia CUDA 生態系統的護城河。
```

**特點**：
- 直接陳述事實與分析結論
- 使用因果邏輯（"基於...考量"）
- 自然提及競爭優勢（"護城河"）但不說"理論"
- 無理論術語

#### ❌ 不好的範例

```markdown
根據 Porter 五力模型分析，AMD 在 AI GPU 市場面臨來自
Nvidia 的強大競爭壓力。我們運用供需分析框架評估市場
動態，發現買方議價能力正在上升。依據護城河理論，
Nvidia 的 CUDA 生態系統構成強大的轉換成本壁壘。
```

**問題**：
- 明確提及理論名稱（Porter、護城河理論）
- 使用方法論術語（"運用...框架"）
- 過於學術化，非自然語言

## 📋 Part A vs Part B 內容分工表

### 完整分工矩陣

| 內容類型 | Part A（深度分析） | Part B（快速參考） | 附錄 |
|---------|------------------|------------------|------|
| **報告標題** | ✅ 完整標題 | ❌ 不重複 | - |
| **評等與目標價** | ✅ 開頭簡述 | ✅ 表格化呈現 | - |
| **重要訊息** | ✅ 80-120字執行摘要 | ❌ 不重複 | - |
| **當季財報分析** | ✅ 詳細論述（300-500字） | ✅ 焦點提煉（4點） | - |
| **長期展望** | ✅ 詳細論述（300-500字） | ✅ 包含在焦點4點中 | - |
| **重大事件** | ✅ 詳細論述（300-500字） | ✅ 包含在焦點4點中 | - |
| **投資建議** | ✅ 詳細估值（250-300字） | ❌ 不重複 | - |
| **投資風險** | ✅ 上行+下行各3-6項 | ❌ 不重複 | - |
| **數據來源** | ✅ 簡版聲明（100字內） | ✅ 詳版說明（200字內） | ✅ 完整引用清單 |
| **參考文獻** | ❌ 不列出 | ❌ 不列出 | ✅ 學術格式引用 |

### 關鍵原則

1. **Part A 注重深度**
   - 詳細論述，提供完整分析邏輯
   - 每個子章節 300-500 字
   - 投資建議 250-300 字

2. **Part B 注重速度**
   - 表格化呈現，便於快速查閱
   - 焦點內容恰好 4 點，每點 60-80 字
   - 濃縮 Part A 核心論點

3. **附錄注重可追溯性**
   - 完整記錄所有數據來源
   - 學術格式引用
   - 包含查詢時間與數據截止日期

## 🎨 自然語氣指導原則

### 一、直接陳述，避免過渡語

#### ❌ 避免使用的冗餘過渡語

```yaml
過度學術化:
  ❌ "值得注意的是..."
  ❌ "需要強調的是..."
  ❌ "有趣的是..."
  ❌ "顯而易見的是..."

過度謹慎:
  ❌ "儘管如此..."
  ❌ "然而另一方面..."
  ❌ "與此同時..."

觀察視角:
  ❌ "從數據中可以看出..."
  ❌ "分析顯示..."
  ❌ "研究表明..."
```

#### ✅ 推薦的自然表達

```yaml
直接陳述:
  ✅ "公司營收增長率達到 25%"
  ✅ "AI GPU 業務成為主要增長動能"
  ✅ "客戶集中度風險值得關注"

自然過渡:
  ✅ "同時，雲端業務也保持穩定增長"
  ✅ "更重要的是，軟體能力正在快速提升"
  ✅ "然而，Nvidia 競爭壓力仍然存在"
```

### 二、使用條件句表達因果

#### ✅ 推薦的因果表達

```markdown
若 OpenAI 合作超出預期，AMD AI GPU 營收可能在 2027 年達到
US$400 億，而非目前預估的 US$310 億。

若 Intel 在 18A 製程成功翻身，AMD 在 x86 CPU 市場的市佔率
增長速度可能放緩。
```

**特點**：
- 使用「若...則...」結構
- 明確量化影響
- 邏輯清晰

### 三、用事實支撐觀點

#### ✅ 推薦的論證結構

```markdown
結構: [論點] + [數據支撐] + [機制說明]

範例:
AMD 資本運用效率優異。過去三年資本支出從 US$50 億增至
US$120 億，投資回報率達到 18%，高於產業平均的 12%。
主要原因是公司 chiplet 架構降低了製程轉換成本，同時
與台積電的長期合作確保了產能優先權。
```

### 四、適當使用類比或比喻

#### ✅ 推薦的類比表達

```markdown
AMD 當前處境類似於 2015-2017 年的狀態：技術路線正確但
執行力需要時間驗證。不同的是，這次 AI 市場的成長速度
更快，留給 AMD 的時間窗口更短。

Nvidia 的 CUDA 生態系統就像 iOS 對蘋果的價值，一旦開發者
習慣了這個平台，轉換成本極高。
```

## 📐 格式規範

### 中英文標點符號使用

```yaml
中文內容:
  句號: 。（全形）
  逗號: ，（全形）
  引號: 「」（全形）
  括號: （）（全形）

英文內容:
  句號: . (半形)
  逗號: , (半形)
  引號: "" (半形)
  括號: () (半形)

混合內容（中文為主）:
  - 中文句子中的英文單詞: 保持半形
  - 數字與單位: US$50 億（數字與貨幣符號間無空格，數字與中文單位間有空格）
  - 百分比: 25%（數字與%間無空格）
```

## 🔍 範例對比

### 完整段落範例

#### ✅ 優秀範例（符合所有原則）

```markdown
AMD 在 AI GPU 市場取得突破性進展，2024 年 AI GPU 營收達
US$50 億，從 2023 年的幾乎零營收實現跨越式增長。公司與
OpenAI 建立的戰略合作夥伴關係成為重要轉折點，預期從
2026 下半年開始供應 MI450 系列 GPU 產品。分析師將 AI GPU
營收預測大幅上調一倍，預估 2025 年達 US$70 億、2026 年
US$140 億、2027 年達 US$310 億。

OpenAI 合作案的戰略意義在於驗證 AMD 作為 Nvidia 之外可行
替代方案的能力。大型雲端服務商基於供應鏈韌性與議價能力
考量，積極尋求第二供應來源。然而，AMD 仍需大幅提升其
ROCm 軟體平台的成熟度，才能真正挑戰 Nvidia CUDA 生態系統
的護城河。AI 模型從 CUDA 遷移至 ROCm 需要投入大量工程
資源，這是 AMD 當前面臨的最大障礙。
```

**優點分析**：
- ✅ 零理論術語
- ✅ 數據完整（具體數字、時間點）
- ✅ 邏輯清晰（現象→原因→展望→風險）
- ✅ 自然語言，易於閱讀
- ✅ 格式正確（貨幣、百分比、標點）

#### ❌ 不良範例（違反多項原則）

```markdown
根據 Porter 五力模型分析，AMD 在 AI GPU 市場面臨來自
Nvidia 的強大競爭壓力。值得注意的是，我們運用供需分析
框架評估市場動態，發現買方議價能力正在上升。依據護城河
理論，Nvidia 的 CUDA 生態系統構成強大的轉換成本壁壘。
從數據中可以看出，AMD 需要大幅提升軟體能力。
```

**問題分析**：
- ❌ 明確提及理論名稱（Porter、護城河理論）
- ❌ 使用方法論術語（"運用...框架"）
- ❌ 冗餘過渡語（"值得注意的是"、"從數據中可以看出"）
- ❌ 過於學術化
- ❌ 缺乏具體數據支撐
